# Создание и настройка учетных записей {#account_creation_and_settings}

import ImageContainer from "../../src/components/ImageContainer";
import {Add} from "@mui/icons-material";

:::info
Для создания и настройки учетных записей пользователь должен обладать правом «Доступ к разделу "Пользователи"». Подробнее см. в разделе [Права доступа](access_rights.mdx).
:::

Чтобы создать и настроить учетную запись:

1.  С главной страницы перейдите на страницу «Пользователи».
    <ImageContainer src={"/images/account_management/accounts_page.png"} description={"Интерфейс страницы «Пользователи»"}/>

2.  Нажмите «<Add/>».
    <ImageContainer src={"/images/account_management/account_create.png"} description={"Страница создания нового пользователя"}/>

3.  Заполните данные о пользователе в полях:
    1. «Имя».

    2.  «Фамилия».

    3.  «Адрес электронной почты» — будет использоваться как логин для входа.

    4.  При необходимости измените язык интерфейса для пользователя.

    5.  «Описание» — отображается на странице «Пользователи».

    6.  «Пароль».

        :::caution
        Пароль должен состоять из 8 символов, иметь заглавную букву, цифру и специальный символ.
        :::

    7.  «Подтвердите пароль».

4.  В блоке «Права и доступ» выберите тип пользователя и настройте права доступа:

    1.  Раскройте блок «Права и доступ».
        :step_result: откроется блок для настройки прав и доступа.

        <ImageContainer src={"/images/account_management/account_create_access_and_rights.png"} description={"Блок «Права и доступ» в настройках пользователя"}/>

    2.  При необходимости деактивируйте учетную запись. Для этого выключите опцию «Активен».

    3.  Включите опцию «Системный администратор», чтобы дать пользователю все права и доступ ко всем местам.

        :::danger
        Будьте осторожны при включении этой опции. Системный администратор имеет доступ ко всем функциям Системы.
        :::

        :::info
        Включение опции «Системный администратор» доступна только другим системным администраторам.
        :::

    4.  Выберите тип пользователя:

        -   «Оператор» — для настройки пользователя как оператора очереди. При отсутствии других
            прав доступа имеет доступ только к просмотру очередей и точкам обслуживания.
        -   «Администратор» — для настройки пользователя как любую другую роль. При отсутствии других
            прав доступа обладает правами Оператора и имеет доступ к настройке графика сотрудников, выходных дней, маячков.
            :::tip
            Тип пользователя определяет возможность настройки модулей места. Тип «Оператор» подходит,
            если пользователь будет работать только с позициями клиентов и не будет администрировать систему.
            В остальных случаях лучше использовать тип «Администратор».
            :::

    5.  Установите права доступа в блоках «Действия с позицией» и «Доступ к разделам».
        Подробнее о правах см. в разделе [Права доступа](access_rights.mdx).

        :::info
        Если вы не являетесь Системным администратором, то можете предоставлять только те права, которые уже назначены вашему аккаунту.
        :::

5.  Выберите места, доступные пользователю:

    1.  Раскройте блок «Места».
        <ImageContainer src={"/images/account_management/account_create_shops.png"} description={"Блок «Места» в настройках пользователя"}/>

    2.  В выпадающем списке выберите место, к которому нужно дать доступ пользователю.

        :::info
        Если вы не являетесь Системным администратором, то можете выбирать только те места, которые уже назначены вашему аккаунту.
        :::

    3.  Нажмите «Добавить место».

    4.  Добавьте все необходимые места.

6.  Нажмите «Сохранить».
    :step_result: пользователь появится в списке на странице «Пользователи», теперь можно
    авторизоваться с электронной почтой и паролем, указанными при создании.
