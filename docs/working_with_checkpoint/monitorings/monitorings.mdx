# Виды мониторингов очереди и точек обслуживания {#monitoring_info}

Основная работа с клиентами и позициями, которые создаются при записи, проводится в мониторинге. Мониторинг существует для каждой отдельной точки обслуживания и для всей очереди. В мониторинг точки обслуживания попадают все позиции, которые были назначены в конкретную точку. В мониторинге очереди отображаются все позиции, даже те, которые не назначены ни на одну из точек обслуживания или не могут быть обслужены, например если единственная оказывающая нужную услугу точка обслуживания выключена.

В панели администрирования представлено несколько страниц мониторинга очереди и точек обслуживания:
-   Для очередей с алгоритмом расчета очереди — «По одному» и «Маршрутная очередь»:
    -   [Планировщик очереди](today_planning.mdx) — представление мониторинга очереди в виде временного графика, в котором наглядно видна каждая точка обслуживания и позиции в них.
    -   [Мониторинг очереди](line_monitoring.mdx) — классическое представление очереди в виде списка со всеми позициями, которые находятся на данный момент в очереди во всех точках обслуживания вместе с позициями, которые не могут быть обслужены или не назначены точке обслуживания.
    -   [Мониторинг точек обслуживания](line_state_monitoring.mdx) — список точек обслуживания и их текущие статусы занятости. Обычно используются администраторами очереди для контроля и выявления проблемных точек обслуживания.
    -   [Мониторинг конкретной точки обслуживания](checkpoint_monitoring.mdx) — список позиций для одной точки обслуживания. Обычно используется сотрудником-оператором точки обслуживания, в которой ведется основная работа по обслуживанию клиентов.

:::info
Настроить алгоритм расчета очереди можно в параметрах очереди, подробнее см. в разделе [Настройка основных параметров очереди](../../lines_management/line_settings/main_settings.mdx).
:::
